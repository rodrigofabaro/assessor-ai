# Student Pages Help

## `/admin/students` (admin student management)

### Purpose
Manage student records used for submission linking.

### Main actions
- search students
- create/edit student
- import students
- inspect submission linkage context

### How to use
1. Search existing student first.
2. Add missing student only when required.
3. Ensure identifiers (email/external ref) are correct for triage matching.

## `/students/[id]` (student cockpit)

### Purpose
View one student profile and their submission/grade history.

### Typical use
1. Open student.
2. Review recent submissions and latest grade outcome.
3. Drill into specific submission detail for audit and feedback.
